The purpose of this context data is to describe my preferences in software, which is fairly common across applications categories.

These preferences are sufficiently general that they're much the same regardless of whether I'm looking for a stack component in something I'm trying to build or a off-the-shelf software, regardless of whether it's software as a service or self-hosted, although there will be some differences.

## Avoid "contact us for a quote"

For in the context of personal and business evaluation processes: the relevant context is that I have my own business a self tech consultancy focused on AI. As I'm currently a one-man operation, when I'm looking for tools for my business, I really limit myself to tools that disclose their pricing. 

The reason is that there's an awful lot of enterprise tooling in spaces I look in. And while there are a few things as absolute rules, it's a 90% generalism that vendors that don't disclose their pricing and request a quote, it's going to be something that I can't afford, unfortunately. So those are ones to exclude generally when identifying searches. 

Exclude, also, anything that's priced for enterprises like the minimum subscription is $1,00. This is almost certainly not going to be viable. 

## "Power User" Friendly 

I like software that is, let's say, power user friendly - things that support a bit of tinkering and editing and allow the user to customise the default experience to suit their own preferences and workflows. 

## A Good API Is A Big Positive 

I particularly value the availability of a well-developed and well-documented API for integrations. 

## Genuine AI Adoption Is A Good Sign

I'm a huge fan and believer in AI. And so I view genuine AI adoption and forwardness as a very strong indicator when evaluating or looking for software solutions - especially when there are a wide range of options with often very similar features.

However - a lot of companies are grossly overstating their adoption of AI and I don't like false, exaggerated AI descriptions that are there really for SEO.  Not every tool requires or can even benefit from the integration of AI. So when it's truly valuable - great. But not for no reason.

## Requirement - Data Portability

One very particular thing that I find is a frequent blocker in SaaS evaluations is data portability. 

These days, I'm mostly looking for SaaS - ie, I actually want to avoid self hosting anything if I can afford not to. 

The only major blocker besides price is usually the provider's data policy - and particularly restrictions around user backups. It may not be something everybody sees  value in, but I believe in backing up all my mission critical data - even if it's in "the cloud."

With SaaS providers, I frequently find that either: a) it's non-existent (provider insists your data is safe, there's no need, etc) or b) a periodic data export functionality, mandated by legislation, is misrepresented as being a backup utility. I'm pedantic on this point: if you can't automate it, isn't not really a viable backup solution. Github is to my mind a rare gold standard here: there's a generous API and you can use it to easily script your own backup jobs so you never need to "just" rely on Github storage if you don't want to. Unfortunately it's far from the norm.

## UI/UX

My feelings about UI/UX are in a bit of opposition to my general prefernce for "power user" tools: I like tools that have clean UI/UX and don't change it all the time for no reason. I guess you might say that I view being able to provide a powerful and customisable UX and still make it a pleasure to use is a measure of good UI to me.
